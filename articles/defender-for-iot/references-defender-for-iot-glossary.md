# <a name="defender-for-iot-glossary"></a>適用於 IoT 詞彙的 Defender

本詞彙提供適用于 IoT 平臺的 Azure Defender 重要詞彙和概念的簡短描述。 選取 [ **深入瞭解** ] 連結，以移至詞彙中的相關詞彙。 這可協助您更快速地學習和使用產品工具。

## <a name="a"></a>A

| 詞彙 | 描述 | 深入了解 |
|--|--|--|
| **存取群組** | 藉由建立存取群組規則，支援大型組織的使用者存取需求。<br /><br />規則可讓您在相關的商務單位、區域、網站和區域上，控制特定使用者角色之 Defender for IoT 內部部署管理主控台的查看和設定存取權。<br /><br />例如，允許來自 Active Directory 群組的安全性分析師存取西歐汽車資料，但防止存取非洲的資料。 | **[內部部署管理主控台](#o)** <br /><br />**[業務單位](#b)** |
| **存取權杖** | 產生存取權杖以存取 Defender for IoT REST API。 | **[API](#a)** |
| **認可警示事件** | 指示 Defender 讓 IoT 針對偵測到的事件隱藏一次警示。 如果再次偵測到事件，則會再次觸發警示。 | **[](#a) <br /> 警示 <br />[學習警示事件](#l) <br /> <br />[靜音警示事件](#m)** |
| **警示** | 適用于 IoT 引擎的 Defender 觸發的訊息，與授權的網路行為、網路異常或可疑的網路活動和流量偏差有關。 | **[轉送規則](#f) <br /> <br />[排除規則](#e) <br /> <br />[系統通知](#s)** |
| **警示批註** | 安全性分析師和系統管理員在警示訊息中提出的批註。 例如，警示批註可能會提供有關要採取之緩和措施的指示，或與事件相關的個人名稱。<br /><br />正在審核警示的使用者可以選擇最能反映事件狀態的批註或批註，或調查警示所採取的步驟。 | **[警示](#a)** |
| **異常引擎** | 適用于 IoT 引擎的 Defender，可偵測不尋常的機器對機器 (M2M) 通訊和行為。 例如，引擎可能會偵測到過多的 SMB 登入嘗試。 偵測到這些事件時，會觸發異常警示。 | **[適用于 IoT 引擎的 Defender](#d)** |
| **API** | 允許外部系統存取 Defender 針對 IoT 探索到的資料，並透過 SSL 連線使用外部 REST API 來執行動作。 | **[存取權杖](#a)** |
| **攻擊向量報表** | 可利用的可攻擊端點之弱點鏈的即時圖形表示。<br /><br />報告可讓您評估風險降低活動在攻擊順序中的影響，以判斷。 例如，您可以評估系統升級是否會中斷攻擊鏈，或是否保留替代的攻擊路徑，以中斷攻擊者的路徑。 這會排定補救和緩和活動的優先順序。 | **[風險評量報告](#r)** |

## <a name="d"></a>D

| 詞彙 | 描述 | 深入了解 |
|--|--|--|
| **資料採礦** | 針對您的網路裝置產生全方位且細微的報告：<br /><br />- **SOC 事件回應**：即時報告以協助處理立即事件回應。 例如，報表可以列出可能需要修補的裝置。<br /><br />- **辯論**：根據調查報表的歷程記錄資料進行報告。<br /><br />- **IT 網路完整性**：有助於改善整體網路安全性的報表。 例如，報表可以列出具有弱式驗證認證的裝置。<br /><br />- **可見度**：涵蓋所有查詢專案的報表，以查看您網路的所有基準參數。<br /><br />儲存資料-讓唯讀使用者查看的報表。 | **[](#b) <br /> 基準 <br />[報表](#r)** |
| **適用于 IoT 引擎的 Defender** | Defender for IoT 中的自我學習分析引擎免除了更新簽章或定義規則的需求。 引擎會使用 ICS 專屬的行為分析和資料科學，針對異常、惡意程式碼、操作問題、通訊協定違規和基準網路活動之間的差異，持續分析網路流量。<br /><br />當引擎偵測到偏差時，就會觸發警示。 您可以從 [ **警示** ] 畫面或 SIEM 來查看和管理警示。 | **[警示](#a)** |
| **適用于 IoT 的 Defender 平臺** | 針對 IoT 感應器和內部部署管理主控台安裝在 Defender 上的 Defender for IoT 解決方案。 | **[](#s) <br /> 感應器 <br />[內部部署管理主控台](#o)** |
| **裝置對應** | Defender for IoT 所偵測網路裝置的圖形標記法。 它會顯示裝置之間的連線，以及每個裝置的相關資訊。 使用對應來：<br /><br />-取得及控制重要的裝置資訊。<br /><br />-分析網路磁區。<br /><br />-匯出裝置詳細資料和摘要。 | **[Purdue 圖層群組](#p)** |
| **裝置清查-感應器** | 裝置清查會顯示 Defender 針對 IoT 所偵測到的廣泛裝置屬性範圍。 您可以使用下列選項：<br /><br />-篩選顯示的資訊。<br /><br />-將此資訊匯出至 CSV 檔案。<br /><br />-匯入 Windows 登錄詳細資料。 | **[Group](#g)** <br /><br />**[裝置清查-內部部署管理主控台](#d)** |
| **裝置清查-內部部署管理主控台** | 您可以從裝置清查中的內部部署管理主控台，查看來自連線感應器的裝置資訊。 這可讓內部部署管理主控台的使用者全面瞭解所有網路資訊。 | **[裝置清查-感應器](#d) <br /> <br />[裝置清查-資料](#d)整合器** |
| **裝置清查-資料整合器** | 內部部署管理主控台的資料整合功能，可讓您使用其他企業資源的資訊，增強裝置清查中的資料。 範例資源包括 Cmdb、DNS、防火牆和 Web Api。 | **[裝置清查-內部部署管理主控台](#d)** |

## <a name="e"></a>E

| 詞彙 | 描述 | 深入了解 |
|--|--|--|
